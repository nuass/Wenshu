# Promotion — 推广策略

## 内容推广工具

### WechatSync（主力推文）
- 浏览器扩展，一次写好 → 同步到知乎、公众号、掘金、CSDN、小红书
- 支持 MCP + Claude Code，可在终端直接发布
- 适合：技术文章、产品介绍、用户案例
- 需要桌面 GUI（Linux Desktop / Mac / Windows）

### MultiPost（扩展平台）
- 浏览器扩展，覆盖抖音、微博图文等 WechatSync 未覆盖的平台
- 有 RESTful API，可做自动化集成
- 与 WechatSync 有重叠，选一个为主力

### social-auto-upload（视频专用）
- Python + Playwright，支持抖音、B站、视频号、快手
- 纯视频工具，不支持文章
- 需要先有飞书答题卡变色的 demo 视频才用得上

## 推广路径

### 阶段一：技术社区冷启动

**目标：** 获取开发者关注 + GitHub Stars

```
写技术文章（知乎 + 掘金 + 公众号）
   ↓
文章主题：
  1. "用飞书 + LLM 给学生建持久记忆：Wenshu 的架构设计"
  2. "为什么不用向量数据库？教学记忆用 Markdown 文件就够了"
  3. "PDF 题库 → 飞书互动卡片：一套 OCR+LLM 的完整 Pipeline"
   ↓
引流到 GitHub
```

**工具：** WechatSync（一键同步多平台）

### 阶段二：教师群体渗透

**目标：** 获取前 10 个付费老师

```
渠道：
  - 微信教育群（独立家教圈子）
  - 小红书（家长 + 老师重叠用户）
  - 飞书官方教育社区
   ↓
内容：
  - 用户故事："一个家教老师如何用文殊管理 30 个学生"
  - 截图/短视频：答题卡变色 + 错题分析 + 学习报告
   ↓
策略：前 3 个老师免费用，换真实反馈和案例
```

**工具：** WechatSync（文章）+ social-auto-upload（视频，有 demo 后）

### 阶段三：开源爆发

**目标：** 开源后快速获取 Stars 和传播

```
时机：有 3+ 真实用户案例后再开源
   ↓
开源当天：
  - GitHub README 完善（中英文）
  - 发 Product Hunt
  - 掘金 + 知乎发"我开源了一个 AI 家教系统"
  - V2EX / 少数派 同步
   ↓
维护：
  - 每月一篇技术更新文章
  - Issue 回复速度 < 24h（前期）
```

## 关键原则

- **先客户后开源**：有真实用户再开源，避免竞争者抢先做托管服务
- **技术叙事 > 功能列表**：讲架构决策（为什么 Markdown、为什么飞书），不只列 feature
- **数据说话**：收集真实数据（答题数、准确率提升、老师省时），用数字说服
- **飞书是渠道，文殊是引擎**：定位清晰，不要把自己定义成"飞书插件"
